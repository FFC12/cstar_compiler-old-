C* Programlama Dili

C*; C ve C++'ın sistem programlama gücünü, modern ownership, typestate, explicit allocator ve thread-safe shared pointer modeliyle daha okunur bir yüzeye taşımayı amaçlayan bir dildir.

LLVM tabanlıProgramlar LLVM IR'a iner, sonra Clang backend ile native binary üretilir.
Ownership odaklı`^` unique pointer, `*` atomic shared handle, `move`, `:=` ve `nomove` ile açık akış.
Typestate`protocol` değerlerin hangi güvenli durumda olduğunu derleme zamanında ifade eder.
Sistem diliAllocator, destructor, async boundary ve thread-safety maliyeti görünür tasarlanır.

Dil Felsefesi

C* sade bir C alternatifi değil; kaynak yönetimini, thread boundary geçişlerini ve state güvenliğini dilin doğal gramerine alan modern bir sistem dili tasarımıdır.

Açık maliyet

Atomic shared count, async boundary, allocator seçimi ve unsafe cast gibi maliyetli kararlar kodda görünür olmalıdır.

Akıllı ama saklamayan

Derleyici pointer lifetime ve typestate konusunda yardımcı olur; gizli policy hook veya beklenmeyen runtime dispatch üretmez.

C++ kadar güçlü

Dil; RAII, move semantics, custom allocator, async/task ve thread safety hedeflerini daha okunur bir yüzeye taşır.

İlk Program

C* programları C benzeri bloklar ve `ret` ile okunur. Fonksiyon dönüş tipi `::` ile yazılır.

main() :: int32 {
    ret 0;
}

Dönüş tipi yazılmazsa fonksiyon `void` kabul edilir. `main() :: int32` process exit code üretmek için önerilen biçimdir.

log_done() :: void {
    ret;
}

main() :: int32 {
    log_done();
    ret 0;
}

Değişkenler ve Atama

Değişken deklarasyonu açık tip ile başlar. Lokal primitive değerler initializer alabilir; initializer yoksa primitive çekirdekte zero-init davranışı tanımlıdır.

int32 x = 7;
int32 y;
y = x + 3;
y += 2;
y <<= 1;

Global değişkenler ve visibility/storage yüzeyi de dilin parçasıdır.

static int32 counter = 0;
import puts(constptr char*) :: int32;
export main() :: int32 { ret 0; }

Native import bildirimi ABI için yalnızca tipi ister; parametre adı okunabilirlik için opsiyoneldir.

import abs(int32) :: int32;
import abs(int32 value) :: int32;

Fonksiyonlar

Fonksiyonlar C benzeri parametre listesiyle tanımlanır. Dönüş tipi `::` sonrasında gelir.

add(int32 a, int32 b) :: int32 {
    ret a + b;
}

main() :: int32 {
    ret add(1, 2);
}
C*'ta fonksiyon parametreleri yalnızca tip kontrolü değil, ownership transfer sözleşmesi de taşır. `int32^` by-value parametre açık `move` ister; `int32*` shared parametre atomic retain yapabilir.

İfadeler ve Operatörler

Arithmetic, comparison ve logical expression'lar C ailesine yakın okunur. Karşılaştırmalar `bool/i1` sonuç üretir ve hedef tipe cast edilebilir.

int32 a = 10 + 20 * 3;
bool ok = a >= 30 && a != 0;
int32 selected = ok ? a : 1;

Unary operator yüzeyi sistem programlama için geniş tutulmuştur.

Kontrol Akışı

`if / elif / else` doğrudan blok üretir. Condition expression integer, float veya pointer değerlerden branch condition'a çevrilebilir.

main() :: int32 {
    if (0) {
        ret 1;
    } elif (1) {
        ret 3;
    } else {
        ret 2;
    }
}

`loop` tasarımı hem while benzeri hem iterable/range benzeri kullanıma ayrılır.

loop(ready) {
    tick();
}

loop(i, value in values) {
    print(i, value);
}

Array

Array boyutları `[]` içinde yazılır; çok boyut için `:` ayracı kullanılır. Tek boyutlu array okuma/yazma ve parametre geçişi çekirdekte çalışır.

write_first(int32[2] arr) :: void {
  arr[0] = 9;
  ret;
}

main() :: int32 {
  int32 values[2] = (4, 5);
  write_first(values);
  ret values[0];
}
Çok boyutlu array semantic'i tasarımın parçasıdır; ilk production modelde indexing/read/write ayrı aşamada tamamlanır.

Pointer ve Reference

C* iki pointer ailesiyle düşünür: `*` shareable pointer, `^` unique ownership pointer. `ref` adres alır, `deref` hedef değeri okur/yazar.

int32 value = 7;
int32* shared = ref value;
int32^ unique = ref value;

deref shared = 8;
*unique = 9;

Reference parametreler çağıranın storage'ına alias olur ve çağrı tarafında açık `ref` ister.

write_ref(int32& value) :: void {
  value = 7;
  ret;
}

main() :: int32 {
  int32 value = 1;
  write_ref(ref value);
  ret value;
}

Ownership

Ownership modeli C*'ın kalbi. `*` paylaşılan handle, `^` tekil sahipliktir. Copy, move ve function boundary davranışı dilden okunur.

Shared `*`

Raw pointer ABI değildir. `{ data, strong }` handle'ı taşır. Copy atomic retain, overwrite release, `move` transfer anlamına gelir.

Unique `^`

Copy yasaktır. Ownership `:=` veya `move` ile aktarılır. Moved source yeniden initialize edilmeden kullanılamaz.

observe(int32* pointer) :: int32 {
  ret strong_count(pointer) as int32;
}

main() :: int32 {
  int32 value = 4;
  int32* source = ref value;
  ret observe(source); // strong_count == 2
}
forward(int32^ pointer) :: int32^ {
  ret move pointer;
}

main() :: int32 {
  int32 value = 9;
  int32^ source = ref value;
  int32^ target = forward(move source);
  ret deref target;
}

`nomove`

`nomove` bir type qualifier değil, parametre seviyesinde ownership-flow kısıtıdır. Parametre okunabilir, fakat fonksiyon gövdesinde tekrar taşınamaz.

read_nomove(nomove int32^ pointer) :: int32 {
  ret deref pointer;
}

// Hatalı:
// int32^ moved := pointer;
// ret move pointer;

Cast

C* safe ve unsafe cast'i ayırır. `cast(expr)` ve `expr as T` safe cast yoludur. Pointer/value kategori geçişleri ve qualifier stripping reddedilir.

int32 a = 4.5 as int32;
int32 b = cast<int32>(5.8);

`unsafe_cast` açıkça daha tehlikeli dönüşümler içindir.

main() :: int32 {
    int32* p = unsafe_cast<int32*>(1);
    if (p) { ret 7; }
    ret 1;
}
`unsafe_cast` dilin kaçış kapısıdır. Kullanımı kod incelemesinde görünür olmalı; implicit pointer/value geçişi tasarım dışıdır.

Struct, Trait ve Enum

C* user-defined type sistemini `struct`, `trait`, `enum` ve `with` ile kurar. Bu bölüm dilin tamamlanmış tasarımını anlatır.

trait Allocator {
    alloc(usize bytes, usize align) :: void*;
    free(void* ptr, usize bytes, usize align) :: void;
}

struct ArenaAllocator with Allocator {
    void* begin;
    usize capacity;
    usize offset;
}

Trait runtime interface değildir; varsayılan yön compile-time capability contract ve static dispatch'tir. Runtime dispatch gerekiyorsa bu açık bir modelle seçilir.

Protocol ve Typestate

`protocol`, bir değerin hangi state'te güvenli olduğunu ifade eder. Eski `policy for T` modeli yerine canonical yön budur.

protocol FileState for FileHandle {
    state closed, opened;

    default closed;

    closed -> opened :: open();
    opened -> closed :: close();

    read() :: !closed;
    write(const char* data) :: !closed;

    scope_exit :: closed;
}

`dynamic protocol` runtime tag gerektiren provability gap durumları içindir. `.=` yalnızca bu açık runtime path için anlamlıdır.

dynamic protocol ConnectionState for Connection {
    state idle, sending, failed;
    idle -> sending :: send();
    sending -> failed :: timeout();
}

conn .= try_send();
if (conn is failed) { break; }

Allocator, Constructor ve Destructor

Allocation bir protocol hook değildir. Explicit allocator trait/object ve constructor/destructor sözleşmesiyle yönetilir.

struct String with Allocator {
    char* buffer;
    usize length;
    usize capacity;

    new(const char* data, Allocator& allocator) :: valid String^ {
        usize n = strlen(data);
        self.buffer = allocator.alloc(n + 1, alignof(char));
        ret self;
    }

    destructor(Allocator& allocator) :: null String^ {
        allocator.free(self.buffer, self.capacity, alignof(char));
        ret self;
    }
}
Allocator, destructor ve protocol birlikte çalışır ama aynı şey değildir: layout struct'ta, allocation allocator'da, lifetime constructor/destructor'da, state safety protocol'dadır.

Async, Await ve Thread Safety

Async/task boundary, ownership boundary olarak kabul edilir. `Send` ve `Sync` runtime vtable değil, trait/capability marker'dır.

trait Send {}
trait Sync {}

consume_on_worker(int32^ owned) async :: int32 {
    ret deref owned;
}

main() :: int32 {
    int32 value = 7;
    int32^ owned = ref value;
    int32 result = await consume_on_worker(move owned);
    ret result;
}
  • `T^` task boundary'den yalnızca açık `move` ile geçer.
  • `T*` boundary'de atomic retain/copy veya açık `move` kullanır.
  • `await` suspension point'tir; ownership checkpoint gibi davranır.

Macro, Attribute ve Directive

Metaprogramming yüzeyi üç katmana ayrılır: attribute compile-time capability/reflectivity, macro kaynak dönüşümü, directive ise function/statement wrapping.

attribute Area<T> {
    area($0, $1) -> T match {
        'Triangle': $0 * $1 / 2,
        'Square': $0 * $1,
        _: cterror("invalid type")
    }
}

macro triangle_area(x, y) {
    x * y / 2;
}

Macro expansion görünür, kaynak konumu izlenebilir ve debug edilebilir olmalıdır. Protocol gizli hook zinciri değildir.

Paket, Import ve Linkage

Paket sistemi `include`, `import`, `export`, `from` ve `as` etrafında tasarlanır.

include involved {
    'ML',
    'X',
    'Y',
}

include "std:math:PI" as PI;

import glClearColor(float r, float g, float b, float a) :: void from "opengl32.lib";

export sin(float64 x) :: float64 from "std:math";

Derleyicinin bugünkü çalışan alt kümesi `import func(...) :: type;` formunu external C çağrısına indirir; `from "lib"` çözümlemesi paket/linker aşamasında tamamlanacaktır.

Standart Kütüphane Yönü

Bugünün çekirdeğinde `print(...)`, `input_int()`, `input_string()`, `clear_screen()`, `flush_output()`, `sleep_ms(ms)`, `read_key()` ve `strong_count(ptr)` gibi built-in'ler vardır. I/O çağrıları compiler içindeki native runtime katmanından CRT/POSIX yüzeyine indirilir; final dilde bu yüzey stdlib/native interop ABI'sine bağlanır.

print("hello\n");
int64 value = input_int();
print(input_string());
clear_screen();
flush_output();
sleep_ms(60);
enable_raw_input();
int32 key = read_key();
disable_raw_input();
int64 count = strong_count(shared);

Kod Stili

İyi C*

  • Bit genişliği kritikse `int32`, `uint64` gibi açık tip yaz.
  • Ownership transferde `move` veya `:=` kullan.
  • Readonly ve constptr ayrımını API sözleşmesinde göster.
  • Unsafe cast'i küçük scope'ta ve gerekçeli kullan.

Kaçınılacaklar

  • Unique pointer'ı plain copy gibi taşımaya çalışma.
  • Protocol'ü allocator veya error hook yerine kullanma.
  • `dynamic protocol` maliyetini gizleme.
  • `nomove` parametreyi başka fonksiyona relay etme.
