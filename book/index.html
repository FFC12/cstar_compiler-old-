C* Language Book

Modern C++ hissini sistem dili disipliniyle birleştiren C*.

C*, açık ownership, thread-safe shared pointer, modül görünürlüğü, struct yaşam döngüsü, native CRT bağlantısı ve LLVM tabanlı üretim akışıyla düşük seviye kontrolü okunabilir bir dil yüzeyinde toplar.

LLVM backend public/private module surface unique/shared ownership constructor/destructor/drop variadic import trait conformance

Felsefe

C* kodu varsayılan olarak açık, yerel ve tahmin edilebilir davranır. Bir sembol public yapılmadığı sürece modül dışına taşmaz; bir değer taşınmadığı sürece sahipliği değişmez; native bir fonksiyon import edilmediği sürece linker yüzeyine girmez. Dilin temel kuralı: maliyet görünür, lifetime belirgin, interop kontrollü.

Stack önce gelir

Constructor var olan storage'ı initialize eder. Allocation ayrı bir operatör/lifecycle yüzeyidir.

Ownership tiptedir

^ unique sahiplik, * atomic shared sahiplik, ref ödünç referans semantiğidir.

Modül yüzeyi nettir

include kaynak görünürlüğü, import/export linker/ABI yüzeyi, public/private erişim kuralıdır.

Söz Dizimi

C* ifade dilinde :: dönüş tipi veya static/module erişimi için, . instance alan/metot erişimi için kullanılır. Satır sonu expression devamını bozmaz; parantez, virgül, operator ve initializer bağlamlarında newline doğal whitespace kabul edilir. 

main() :: int32 {
  int32 a = 10;
  int32 b = 20;
  int32 total = a
              + b;

  if total == 30 {
    ret 0;
  }

  ret 1;
}

Tipler ve Literaller

Primitive tipler explicit genişlik taşır: int8, int16, int32, int64, unsigned karşılıkları, isize, usize, float32, float64, bool, char ve void. C ABI için char* ve const char* raw string yüzeyi korunur. 

main() :: int32 {
  const char* title = "C*";
  char letter = 'A';
  int64 count = 922337203685477580;
  float64 ratio = 0.75;
  bool ok = true;

  ret 0;
}

Fonksiyonlar

Fonksiyon adı parametre listesinden önce gelir, dönüş tipi :: sonrasında yazılır. Named param syntax belgelenir; ABI ve çağrı lowering aşamasında parametre sırası korunur. ret her dönüşü açık hale getirir. 

sum(int32 left, int32 right) :: int32 {
  ret left + right;
}

main() :: int32 {
  int32 value = sum(20, 22);
  ret value - 42;
}

Variadic imza

Variadic parametre ... ile yazılır ve yalnız son parametre olabilir. Bu yüzey özellikle CRT fonksiyonları ve printf/scanf uyumlu standard library için kullanılır. 

import from "std:crt" {
  printf(const char* fmt, ...) :: int32;
}

main() :: int32 {
  printf("answer=%d ratio=%f\n", 42, 0.5);
  ret 0;
}

Modüller

include bir C* dosyasını derleme birimine dahil eder. as alias kullanıldığında erişim module namespace üzerinden yapılır. Varsayılan görünürlük private, dış yüzey public ile tanımlıdır. import ve export visibility değildir; linker ve ABI sözleşmesidir. 

// modules/math.cstar
public static base :: int32 = 10;

public add(int32 value) :: int32 {
  ret base + value;
}

// main.cstar
include "modules/math.cstar" as math;

main() :: int32 {
  ret math::add(32);
}

static modül seviyesinde tek storage/tek sembol davranışıdır. Public static değişkenler alias edilmiş modülden module::name ile okunur ve yazılır.

Standart Kütüphane

std/core.cstar, CRT import yüzeyini C* kodundan kullanılabilir küçük bir çekirdek API'ye bağlar. Print/read işlemleri compiler builtin'i değildir; normal C* modülü olarak include edilir. 

include "../../std/core.cstar" as core;

main() :: int32 {
  core::core_print("name: ");
  char* name = core::core_read_string();

  core::core_print("score: ");
  int32 score = core::core_read_i32();

  core::core_print(name);
  core::core_print(" -> ");
  core::core_print_i32(score);
  core::core_print("\n");
  ret 0;
}

Kontrol Akışı

if / elif / else blokları expression-style karmaşıklık yerine açık blok semantiği taşır. loop sonsuz döngüdür; break ve continue nested bloklarda doğru hedefe lower edilir. 

main() :: int32 {
  int32 i = 0;
  int32 total = 0;

  loop {
    i = i + 1;
    if i == 3 {
      continue;
    } elif i > 6 {
      break;
    }
    total = total + i;
  }

  ret total;
}

Diziler

Tek boyutlu ve çok boyutlu diziler sabit boyut bilgisini tip yüzeyinde taşır. Çok boyutlu initializer, şekli görünür kılan nested tuple notasyonuyla yazılır. 

main() :: int32 {
  int32 values[4] = (1, 2, 3, 4);
  int32 matrix[2:3] = (
    (1, 2, 3),
    (4, 5, 6)
  );

  int32 sum = 0;
  for item in values {
    sum = sum + item;
  }

  ret matrix[1:2] + sum;
}

Sahiplik ve Pointer

C* pointer yüzeyi raw pointer alışkanlığını sahiplik tipleriyle ayırır. ^ unique pointer taşınabilir ama kopyalanamaz; * shared pointer atomic reference count ile thread-safe paylaşılır. move sahiplik transferini, nomove ödünç okuma niyetini ifade eder. 

struct Box {
  int32 value;

  constructor(int32 value) {
    self.value = value;
  }
}

take(Box^ item) :: int32 {
  ret item.value;
}

main() :: int32 {
  Box^ owned = new Box(7);
  int32 a = take(move owned);

  Box* sharedValue = shared new Box(11);
  Box* copy = sharedValue;
  int64 refs = strong_count(sharedValue);

  ret a + copy.value + cast(refs);
}

Struct Yaşam Döngüsü

Struct alan, method, constructor, destructor ve scope-exit drop semantiğini tek modelde toplar. Constructor allocation yapmaz; var olan storage'ı geçerli object haline getirir. Destructor object'in iç kaynaklarını kapatır. Normal kullanıcı kodu destructor'ı method gibi çağırmaz; erken bırakma için drop, olağan yaşam sonu için scope-exit kullanılır. 

struct FileHandle {
  int32 fd;

  constructor(int32 fd) {
    self.fd = fd;
  }

  destructor() {
    if self.fd >= 0 {
      // native close surface burada çağrılır
      self.fd = -1;
    }
  }

  valid() :: bool {
    ret self.fd >= 0;
  }
}

main() :: int32 {
  FileHandle handle = FileHandle(3);
  if handle.valid() {
    drop handle;
  }
  ret 0;
}

Static methodlar tipe aittir ve Type::method() ile çağrılır. Instance alan ve method erişimi . kullanır; :: pointer field erişimi değildir.

Trait ve Operatörler

Trait, bir struct'ın karşılaması gereken method sözleşmesini tanımlar. Conformance açık with yüzeyiyle okunur; dispatch static ve tip güvenlidir. Değer operatörleri struct içinde açık operator methodlarıyla tanımlanır. 

trait Scorable {
  score() :: int32;
}

struct Meter with Scorable {
  int32 value;

  constructor(int32 value) {
    self.value = value;
  }

  score() :: int32 {
    ret self.value;
  }

  operator +(Meter other) :: Meter {
    ret Meter(self.value + other.value);
  }

  operator ==(Meter other) :: bool {
    ret self.value == other.value;
  }
}

main() :: int32 {
  Meter a = Meter(10);
  Meter b = Meter(32);
  Meter c = a + b;
  ret c.score();
}

Native Bağlantı

Native dünya import from bloklarıyla dile alınır. CRT sembolleri "std:crt" kaynağından, platform kütüphanesi sembolleri literal library adıyla import edilir. Export ise C* fonksiyonunu dış linker yüzeyine açar. 

import from "std:crt" {
  abs(int32 value) :: int32;
  printf(const char* fmt, ...) :: int32;
}

export distance(int32 value) :: int32 {
  ret abs(value);
}

Normal executable üretiminde kullanılmayan export yüzeyi diagnostic verebilir; library emit akışında export edilen semboller ABI sözleşmesinin parçasıdır.

Diagnostic Modeli

Compiler, dil tasarımına ait kuralları parse sonrasında semantic analizde doğrular: private sembolün modül dışından kullanımı, variadic parametrenin son sırada olmaması, moved/dropped değerin tekrar kullanımı, trait conformance eksikleri, shared/unique pointer assignment kuralları ve static erişim hataları açık hata mesajları üretir. 

struct Token {
  int32 id;
  constructor(int32 id) { self.id = id; }
}

main() :: int32 {
  Token t = Token(1);
  drop t;

  // Burada t artık dropped state'tedir; sonraki kullanım diagnostic üretir.
  ret t.id;
}

C* için “çalıştı ama belirsizdi” kabul edilebilir bir hedef değildir. Dil, güvenli sistem programlama için davranışı görünür yapar ve yanlış lifetime/visibility kullanımını erken durdurur.

C* kitabı, dilin referans yüzeyini tek bir okuma akışında toplar: syntax, modül sistemi, native bağlantı, ownership, struct lifecycle, trait conformance ve diagnostic modeli aynı tasarım çizgisinin parçalarıdır.
